# AGENT_DOCS

## Mission
Garder la documentation à jour et alignée avec le code et les agents.

## Responsabilités
- Maintenir `agent.md`, `codex_log.md`, `README.md` et l'arborescence `docs/` (index, specs, roadmap, ADR).
- Coordonner la mise à jour des sous-agents lorsque les spécifications changent.
- Vérifier la cohérence entre documentation, CI et code livré.

## Processus
- Toute évolution fonctionnelle/technique doit être décrite dans `docs/specs_functionnelles.md` et/ou `docs/specs_techniques.md` avant implémentation.
- Les décisions structurantes sont consignées dans `docs/decisions.md`.
- Ajouter une entrée dans `codex_log.md` pour tracer les évolutions documentaires et leur impact.
